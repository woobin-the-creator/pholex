⚠ 옵션A 확정 전 아직 안 정해진 가지

  1. 실시간 갱신(질문2) — WS 완전 제거 vs 30분 dump 완료 시 push vs 프론트 N분 폴링.
  2. 저장 의미론 — "일괄 저장"이 전체 교체(내 행 삭제 후 재삽입)인지 증분 upsert인지. 중복 lot_id·빈 row 처리.
  3. 존재하지 않는 lot_id — 유저가 입력한 lot_id가 lots 테이블에 없을 때: 저장 거부 / 저장은 하되 "unknown" 표시 / dump 후 자동 매칭.
  4. "hold" 개념 존치 — 섹션명이 "내 lot hold"인데, 이제 run/wait/hold 무관하게 모니터링인지, 여전히 hold 상태만 의미 있는지.
  5. lots 테이블 스키마 = 사내AI 계약 — readonly로 보여줄 컬럼(status·step·equipment·…)을 dump 잡과 어떻게 합의할지.
